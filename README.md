# Work Notes

이 저장소는 의료 / 헬스케어 소프트웨어 시스템을 개발하며
실제 업무 환경에서 겪은 이슈와 구조적 변경 사항을
엔지니어링 관점에서 기록한 노트입니다.

단순히 사용한 기술을 나열하는 것이 아니라,
**문제가 어떻게 발생했고,
어떻게 분석했고,
어떻게 해결했으며,
어떻게 검증했는지**에 초점을 둡니다.

---

## 이 저장소의 목적

- 실제 서비스 환경에서 발생한 **실무 이슈 기록**
- 장애, 구조 변경, 설계 판단에 대한 **엔지니어 관점 정리**
- 의료 도메인의 특성(보안, 데이터 무결성, 추적성)을 고려한 사고 방식 축적
- 개인 학습이 아닌 **재사용 가능한 커리어 자산** 생성

이 저장소는 튜토리얼이나 예제 모음이 아닙니다.

---

## 구조 설명

각 주제 폴더에는 다음과 같은 파일이 포함될 수 있습니다.

- `README.md`  
  → 해당 이슈의 전체 요약 및 핵심 메시지
- `trouble.md`  
  → 실제로 발생한 문제, 증상, 원인, 해결 과정
- `architecture.md`  
  → 시스템 구조, 설계 변경, 아키텍처 관점 정리
- `tests.md`  
  → 수정 이후 검증 방법 및 회귀 테스트 시나리오

모든 주제가 모든 파일을 가질 필요는 없습니다.
